# Git Workflow

## Introduction

A Git workflow defines how developers collaborate using Git.

## Common Steps

1. Clone the repository.
2. Create a new branch.
3. Make changes.
4. Commit changes.
5. Push the branch.
6. Create a Pull Request.
7. Merge into the main branch.

## Best Practices

- Keep commits small.
- Pull before pushing.
- Use meaningful commit messages.

## Interview Question

**Q:** What is a Git workflow?

**A:** It is a standard process followed by developers while working with Git.
