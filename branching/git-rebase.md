# Git Rebase

## Introduction

The `git rebase` command moves or reapplies commits to create a cleaner commit history.

## Syntax

```bash
git rebase main
```

## Example

```bash
git rebase main
```

## Real World Use Case

Update your feature branch with the latest changes from the main branch.

## Best Practices

Avoid rebasing branches that are already shared with others.

## Interview Question

**Q:** What is the difference between Merge and Rebase?

**A:** Merge preserves history, while Rebase creates a linear history.
