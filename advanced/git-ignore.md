# .gitignore

## Introduction

A `.gitignore` file tells Git which files and folders should not be tracked.

## Example

```text
node_modules/
.env
*.log
```

## Real World Use Case

Ignore temporary files, build files, and sensitive information.

## Best Practices

Never commit passwords, API keys, or environment files.

## Interview Question

**Q:** Why is `.gitignore` important?

**A:** It prevents unnecessary or sensitive files from being tracked by Git.
