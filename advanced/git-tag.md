# Git Tag

## Introduction

Git tags are used to mark important points in the repository, such as software releases.

## Syntax

```bash
git tag <tag-name>
```

## Example

```bash
git tag v1.0
git push origin v1.0
```

## Real World Use Case

Mark release versions like v1.0 or v2.0.

## Best Practices

Use semantic versioning (v1.0.0, v1.1.0).

## Interview Question

**Q:** Why do developers use Git tags?

**A:** To identify important release versions.
